# DINA2 Sidechain-Aware Dynamics Pipeline

This repository implements a sidechain-aware DINA2 pipeline for residue-level prediction of microsecond-millisecond protein dynamics.

It is designed to extend a Dyna-1-style embedding model with explicit sidechain structural context from ESMFold PDBs while avoiding the scientific trap of treating one static structure as direct evidence of dynamics. Static structures provide chemical and packing context; experimental labels still come from NMR-derived sources such as RelaxDB-CPMG.

The pipeline is intentionally fail-closed: labels are never joined to features by raw residue number or by `bmrb_id` alone. Every label assignment must pass sequence alignment QC.

## What This Repo Does

- Imports Dyna-1 RelaxDB-CPMG labels using the verified `entry_ID`, `sequence`, and `label` schema.
- Maps labels to manifest entities using strict sequence alignment.
- Detects terminal His-tags (`H{6,}`) and masks tag residues from evaluation.
- Extracts sidechain-aware residue features from ESMFold PDBs.
- Pre-extracts frozen ESM-2 layer-30 embeddings.
- Builds model-ready residue tables.
- Creates protein-level sequence-clustered splits.
- Trains embedding-only, sidechain-only, and ESM-2-plus-sidechain fusion baselines.
- Saves QC reports and normalizer/checkpoint metadata for reproducibility.

## What This Repo Does Not Use

- HSQC image segmentation
- inner ellipse area
- peak area
- raw HSQC image features
- linewidth extracted from images

## Quick Start

```bash
python -m pip install -e ".[dev]"
pytest
```

For structural and ML commands:

```bash
python -m pip install -e ".[structure,ml,dev]"
```

The core label importer and lightweight feature extraction paths only require `numpy` and `pandas`.

## Core Commands

Import Dyna-1 CPMG labels and align them to your ESMFold manifest:

```bash
dina2 import-dyna1-labels \
  --label-zip /path/to/RelaxDB_CPMG_22jan2025.json.zip \
  --manifest "/Users/fangruizhang/Desktop/NMR and research/esmfold_structures_package/esmfold_pdb_manifest.csv" \
  --out-labels data/labels/cpmg_labels.csv \
  --out-qc data/qc/cpmg_alignment_qc.csv
```

Extract sidechain features:

```bash
dina2 extract-sidechain-features \
  --manifest "/Users/fangruizhang/Desktop/NMR and research/esmfold_structures_package/esmfold_pdb_manifest.csv" \
  --pdb-root "/Users/fangruizhang/Desktop/NMR and research/esmfold_structures_package/esmfold_structures" \
  --out data/features/sidechain_features.csv \
  --limit 20
```

Pre-extract ESM-2 embeddings:

```bash
dina2 extract-esm2-embeddings \
  --manifest "/Users/fangruizhang/Desktop/NMR and research/esmfold_structures_package/esmfold_pdb_manifest.csv" \
  --out-dir data/embeddings/esm2_t30_150M_UR50D_layer30 \
  --limit 20
```

Build a model-ready joined table:

```bash
dina2 build-dataset \
  --features data/features/sidechain_features.csv \
  --labels data/labels/cpmg_labels.csv \
  --out data/datasets/cpmg_features.csv
```

## Label Safety Rules

- Terminal His-tags are `H{6,}` at either terminus and are always masked out.
- Label sequence is aligned to manifest sequence, then manifest sequence is aligned to parsed PDB sequence.
- Default acceptance requires `min_label_coverage=0.90` and `min_identity=0.90`.
- Ambiguous, unmapped, low-coverage, or mismatched mappings are written to QC and excluded.
- Multi-entity entries are resolved by sequence alignment unless the label file explicitly carries `entity_id`.

## Notes

The ESMFold PDBs generated by the companion `ESM_pipeline` store confidence in the B-factor column on a 0-1 scale. This package preserves that value as `plddt`.
